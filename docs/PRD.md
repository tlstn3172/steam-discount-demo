## 1. 제품 개요 (Product Overview)

### 1.1 제품명
Steam Value Insights (SVI)

### 1.2 제품 설명
스팀의 방대한 할인 데이터 속에서 단순한 가격 인하율이 아닌, 실제 구매 가치와 기회비용을 분석하여 최적의 구매 의사결정을 돕는 데이터 분석 대시보드입니다.

### 1.3 제품 비전
게이머들이 한정된 예산으로 최대의 즐거움을 얻을 수 있도록, 데이터 기반의 스마트한 구매 의사결정을 지원하는 필수 도구가 되는 것

---

## 2. 기획 배경 및 목적 (Problem Statement & Objectives)

### 2.1 문제 정의
스팀의 잦은 세일과 방대한 게임 수로 인해 사용자는 어떤 게임을 지금 사는 것이 가장 이득인지 판단하기 어려운 정보 과부하 상태에 노출되어 있습니다.

**주요 페인 포인트:**
- 할인율이 높아도 실제로 최저가가 아닌 경우가 많음
- 구매 후 플레이하지 않는 게임이 라이브러리에 쌓임
- 숨겨진 명작 게임을 발견하기 어려움
- 가격 대비 실제 플레이 가치를 판단하기 어려움

### 2.2 프로젝트 목적
1. 가격 데이터와 플레이 시간 데이터를 결합한 가성비 지표 제공
2. 단순 할인율이 아닌 역대 최저가 비교를 통한 구매 시점 제언
3. 데이터 기반의 숨겨진 명작 발굴로 사용자 만족도 제고
4. 합리적인 구매 의사결정을 통한 게임 라이브러리 활용도 향상

---

## 3. 주요 타겟 사용자 (Target Audience)

### 3.1 Primary Persona: 합리적 게이머 "민수"
- 연령: 25-35세
- 특징: 한정된 예산으로 최대의 즐거움을 찾고자 하는 합리적 소비자
- 니즈: 구매 후 후회하지 않을 게임을 찾고 싶음
- 행동 패턴: 세일 기간마다 스팀을 방문하지만 무엇을 살지 결정하기 어려워함

### 3.2 Secondary Persona: 인디 게임 탐색가 "지은"
- 연령: 20-30세
- 특징: 대작(AAA) 게임 외에 자신의 취향에 맞는 고퀄리티 인디 게임을 찾는 탐색가
- 니즈: 마케팅에 묻힌 숨겨진 명작을 발견하고 싶음
- 행동 패턴: 리뷰와 평점을 꼼꼼히 확인하며 새로운 게임을 탐색

### 3.3 Tertiary Persona: 실용적 소비자 "현우"
- 연령: 30-40세
- 특징: 구매 후 플레이하지 않는 게임을 줄이고 실용적 소비를 지향
- 니즈: 실제로 플레이할 가치가 있는 게임만 구매하고 싶음
- 행동 패턴: 구매 전 충분한 조사를 하며 신중하게 결정

## 4. 핵심 기능 (Core Features)

### 4.1 가성비 지수 (VPI, Value-to-Price Index) 산출

**기능 설명:**
게임의 예상 플레이 시간 대비 현재 가격의 효율성을 수치화하여 제공

**계산 수식:**
```
VPI = (예상 플레이 시간 / 현재 가격) × 1000
```

**출력 형식:**
- 1,000원당 즐길 수 있는 시간(시간 단위)으로 표시
- 높은 순으로 정렬하여 순위 제공

**데이터 소스:**
- 플레이 시간: HowLongToBeat API
- 현재 가격: CheapShark API

**우선순위:** P0 (필수)

---

### 4.2 스마트 구매 신호등 (Purchase Signal)

**기능 설명:**
현재 할인가가 역대 최저가(All-time Low)와 얼마나 근접한지 시각적으로 표시

**신호 구분:**
1. **초록 (Best):** 역대 최저가 경신 또는 동일
   - 표시: 🟢 "지금이 최적 구매 시점입니다"
   
2. **노랑 (Good):** 역대 최저가와 10% 이내 차이
   - 표시: 🟡 "좋은 가격입니다"
   
3. **빨강 (Wait):** 이전 할인폭보다 낮거나 곧 정기 세일이 예상되는 시점
   - 표시: 🔴 "더 기다리는 것을 권장합니다"

**데이터 소스:**
- 역대 최저가: CheapShark API
- 가격 히스토리: CheapShark API

**우선순위:** P0 (필수)

---

### 4.3 데이터 기반 틈새 시장 발굴 (Hidden Gem Filter)

**기능 설명:**
마케팅 자본에 가려진 고득점 게임을 필터링하여 추천

**필터링 로직:**
```
조건 1: 긍정적 평가 비율 ≥ 90%
AND
조건 2: 총 리뷰 수 ≤ 1,000개
AND
조건 3: 현재 할인율 ≥ 50%
```

**출력:**
- 조건을 충족하는 게임 리스트
- VPI 지수 순으로 정렬
- "숨겨진 명작" 배지 표시

**데이터 소스:**
- 리뷰 데이터: SteamSpy API
- 할인율: CheapShark API

**우선순위:** P1 (중요)

---

### 4.4 역대 가격 변동 그래프

**기능 설명:**
선택한 게임의 가격 변동 히스토리를 시각화

**표시 정보:**
- 최근 1년간 가격 변동 추이
- 역대 최저가 표시
- 주요 세일 이벤트 마커 (예: 여름 세일, 겨울 세일)

**우선순위:** P1 (중요)

---

### 4.5 필터 및 정렬 기능

**필터 옵션:**
- 장르 (액션, RPG, 인디, 시뮬레이션 등)
- 가격 범위 (₩0-10,000, ₩10,000-30,000 등)
- 할인율 (50% 이상, 75% 이상 등)
- 평가 점수 (긍정적 평가 90% 이상 등)

**정렬 옵션:**
- VPI 지수 높은 순
- 할인율 높은 순
- 가격 낮은 순
- 평가 점수 높은 순
- 구매 신호등 우선순위

**우선순위:** P0 (필수)

---

### 4.6 메타 정보 통합 표시

**표시 정보:**
- 메타크리틱 점수
- 스팀 평가 점수 및 리뷰 수
- 예상 플레이 시간 (메인 스토리, 컴플리트)
- 출시일
- 개발사/퍼블리셔

**우선순위:** P1 (중요)

## 5. 사용자 흐름 (User Flow)

### 5.1 메인 사용 시나리오

```
1. 메인 접속
   ↓
2. 현재 할인 중인 전체 리스트 확인
   ↓
3. 필터 설정 (장르, 예산 범위 등)
   ↓
4. 데이터 정렬 (VPI 지수 또는 구매 신호 우선순위)
   ↓
5. 관심 게임 선택
   ↓
6. 상세 정보 확인 (가격 그래프, 메타 점수 등)
   ↓
7. 구매 결정
   ↓
8. 스팀 스토어 페이지로 이동
```

### 5.2 Hidden Gem 발굴 시나리오

```
1. 메인 접속
   ↓
2. "숨겨진 명작" 필터 활성화
   ↓
3. 추천 리스트 확인
   ↓
4. 게임 상세 정보 탐색
   ↓
5. 찜 목록 추가 또는 즉시 구매
```

## 6. 화면 구성 (UI/UX Requirements)

### 6.1 메인 대시보드
- 상단: 필터 및 정렬 옵션
- 중앙: 게임 카드 그리드 레이아웃
  - 게임 썸네일
  - 게임 제목
  - 현재 가격 및 할인율
  - VPI 지수
  - 구매 신호등
  - 평가 점수
- 우측: 빠른 필터 사이드바

### 6.2 게임 상세 페이지
- 상단: 게임 헤더 (제목, 썸네일, 기본 정보)
- 중앙 좌측: 가격 변동 그래프
- 중앙 우측: 주요 지표 (VPI, 구매 신호, 메타 점수)
- 하단: 상세 정보 및 스팀 스토어 링크

### 6.3 반응형 디자인
- 데스크톱: 그리드 레이아웃 (3-4열)
- 태블릿: 그리드 레이아웃 (2열)
- 모바일: 리스트 레이아웃 (1열)

## 7. 핵심 지표 (Key Performance Indicators, KPI)

### 7.1 기술적 KPI
- **데이터 정확도:** 외부 API와의 데이터 동기화 성공률 ≥ 99%
- **응답 시간:** 페이지 로딩 시간 ≤ 2초
- **데이터 신선도:** 가격 정보 업데이트 주기 ≤ 1시간

### 7.2 사용자 경험 KPI
- **사용자 체류 시간:** 평균 세션 시간 ≥ 5분
- **페이지 뷰:** 세션당 평균 페이지 뷰 ≥ 10
- **재방문율:** 월간 재방문율 ≥ 40%

### 7.3 비즈니스 KPI
- **추천 효율성:** 제안된 게임의 찜 목록 추가율 ≥ 20%
- **전환율:** 스팀 스토어 페이지 클릭률 ≥ 30%
- **사용자 만족도:** Hidden Gem 추천 만족도 ≥ 4.0/5.0

## 8. 기술 스택 및 데이터 소스 (Technical Resources)

### 8.1 데이터 소스

**외부 API:**
1. **CheapShark API**
   - 용도: 가격 정보 및 역대 최저가
   - 업데이트 주기: 1시간마다
   
2. **HowLongToBeat (HLTB)**
   - 용도: 플레이 시간 데이터
   - 업데이트 주기: 주 1회
   
3. **SteamSpy API**
   - 용도: 리뷰 수 및 소유자 추정치
   - 업데이트 주기: 일 1회

### 8.2 기술 스택

**백엔드:**
- Python 3.11+
- FastAPI (API 서버)
- Pandas (데이터 분석)
- APScheduler (정기 데이터 수집)
- PostgreSQL (데이터 저장)

**프론트엔드:**
- React 18+
- TypeScript
- Chart.js (가격 그래프)
- Tailwind CSS (스타일링)

**인프라:**
- Docker (컨테이너화)
- GitHub Actions (CI/CD)

## 11. 제약사항 및 리스크 (Constraints & Risks)

### 11.1 기술적 제약사항
- 외부 API 의존도가 높아 API 변경 시 영향을 받을 수 있음
- API 호출 제한(Rate Limit)으로 인한 데이터 수집 제약
- 플레이 시간 데이터의 정확도 한계

### 11.2 리스크 관리
| 리스크 | 영향도 | 대응 방안 |
|--------|--------|-----------|
| API 서비스 중단 | 높음 | 백업 데이터 소스 확보, 캐싱 전략 |
| 데이터 정확도 문제 | 중간 | 다중 소스 교차 검증 |
| 사용자 트래픽 급증 | 중간 | 캐싱 및 CDN 활용 |
| 법적 이슈 (저작권) | 낮음 | 공개 API만 사용, 이용약관 준수 |

## 13. 전략적 기대 효과

이 프로젝트는 단순한 도구 개발을 넘어, 복잡한 데이터를 가공하여 의사결정 모델을 구축하는 과정을 보여줍니다. 이는 경영전략 기획에서 중시하는 다음 역량을 입증합니다:

1. **데이터 분석 능력:** 다양한 데이터 소스를 통합하여 의미 있는 인사이트 도출
2. **문제 해결 능력:** 사용자의 실제 페인 포인트를 파악하고 해결책 제시
3. **전략적 사고:** 단순 기능 나열이 아닌 사용자 가치 중심의 제품 설계
4. **실행력:** 기획부터 구현까지 전 과정을 주도적으로 수행
