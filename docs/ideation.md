## 제품 요구사항 정의서 (PRD): Steam Value Insights (SVI)

### 1. 제품 개요 (Product Overview)

스팀의 방대한 할인 데이터 속에서 단순한 가격 인하율이 아닌, 실제 구매 가치와 기회비용을 분석하여 최적의 구매 의사결정을 돕는 데이터 분석 대시보드입니다.

### 2. 기획 배경 및 목적 (Problem Statement & Objectives)

* 문제 정의: 스팀의 잦은 세일과 방대한 게임 수로 인해 사용자는 어떤 게임을 지금 사는 것이 가장 이득인지 판단하기 어려운 정보 과부하 상태에 노출됨.
* 프로젝트 목적:
1. 가격 데이터와 플레이 시간 데이터를 결합한 가성비 지표 제공.
2. 단순 할인율이 아닌 역대 최저가 비교를 통한 구매 시점 제언.
3. 데이터 기반의 숨겨진 명작 발굴로 사용자 만족도 제고.



### 3. 주요 타겟 사용자 (Target Audience)

* 한정된 예산으로 최대의 즐거움을 찾고자 하는 합리적 게이머.
* 구매 후 플레이하지 않는 게임(라이브러리 채우기)을 줄이고 실용적 소비를 지향하는 사용자.
* 대작(AAA) 게임 외에 자신의 취향에 맞는 고퀄리티 인디 게임을 찾는 탐색가.

---

### 4. 핵심 기능 (Core Features)

#### 4.1. 가성비 지수 (VPI, Value-to-Price Index) 산출

* 기능: 게임의 예상 플레이 시간 대비 현재 가격의 효율성을 수치화.
* 수식: 
* 비고: 1,000원당 즐길 수 있는 시간 단위로 환산하여 순위 제공.

#### 4.2. 스마트 구매 신호등 (Purchase Signal)

* 기능: 현재 할인가가 역대 최저가(All-time Low)와 얼마나 근접한지 시각화.
* 구분:
1. 초록(Best): 역대 최저가 경신 또는 동일.
2. 노랑(Good): 역대 최저가와 10% 이내 차이.
3. 빨강(Wait): 이전 할인폭보다 낮거나 곧 정기 세일이 예상되는 시점.



#### 4.3. 데이터 기반 틈새 시장 발굴 (Hidden Gem Filter)

* 기능: 마케팅 자본에 가려진 고득점 게임 필터링.
* 로직: 긍정적 평가 90% 이상 AND 리뷰 수 1,000개 이하 AND 할인율 50% 이상의 조건 충족 데이터 추출.

---

### 5. 사용자 흐름 (User Flow)

1. 메인 접속: 현재 할인 중인 전체 리스트 확인.
2. 필터 설정: 사용자가 선호하는 장르 및 예산 범위 설정.
3. 데이터 정렬: VPI 지수 높은 순 또는 스마트 구매 신호 우선순위로 정렬.
4. 상세 확인: 선정된 게임의 메타크리틱 점수와 역대 가격 변동 그래프 확인.
5. 구매 결정: 스팀 스토어 페이지로 이동.

---

### 6. 핵심 지표 (Key Performance Indicators, KPI)

* 데이터 정확도: 외부 API(CheapShark, HLTB)와의 데이터 동기화 성공률.
* 사용자 체류 시간: 단순 가격 확인을 넘어 가성비 지표를 비교 분석하는 시간.
* 추천 효율성: 제안된 명작 리스트의 찜 목록 추가 및 실제 구매 전환 경로 추적.

---

### 7. 기술 스택 및 데이터 소스 (Technical Resource)

* 데이터 소스:
1. CheapShark API (가격 정보 및 역대 최저가)
2. HowLongToBeat (플레이 시간 데이터)
3. SteamSpy API (리뷰 수 및 소유자 추정치)


* 분석 도구: Python (Pandas), 정기적 데이터 수집을 위한 스케줄러.

---

### 8. 전략적 기대 효과

이 프로젝트는 단순한 도구 개발을 넘어, 복잡한 데이터를 가공하여 의사결정 모델을 구축하는 과정을 보여줍니다. 이는 경영전략 기획에서 중시하는 데이터 분석 기반의 인사이트 도출 능력을 입증하는 훌륭한 포트폴리오가 될 것입니다.
